## MainActivity.java 处理点击事件

下一个方法是

```
    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button"
        tools:layout_editor_absoluteX="160dp"
        tools:layout_editor_absoluteY="217dp" 
        tools:ignore="MissingConstraints" />
```

第一个内容中设置了` id` 那么我们Java代码去寻找`id` 从而对部件操作

需要在`onCreate`中找到这个ID

```
Button button = findViewById(R.id.but111111);
```

然后就是为他设置点击事件

```
button.setOnClickListener
```

其内部需要实现一个接口 所以完整代码如下

```
public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        EdgeToEdge.enable(this);
        setContentView(R.layout.activity_main);
        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main), (v, insets) -> {
            Insets systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars());
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom);
            return insets;
        });


        Button button = findViewById(R.id.but111111);
        button.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                Log.d("cccccc","MainActivity.java 处理点击按钮 触发");
            }
        });

    }
}
```

这样就是两个按钮事件的触发 我们可以实验一下 当然可以让匿名函数在外部实现


# 日志输出

我们可以使用`Log` 输出日志相关的 还可以通过`toast` 输出

`toast`是现实给用户 弹出

```
Toast.makeText(context, "MainActivity.java 处理点击按钮 触发", Toast.LENGTH_SHORT).show();

```

`Toast.LENGTH_SHORT` 是显示时间 

并且最后需要调用`.show()` 执行

一般是`context` 就是哪个界面 我们可以传入的是`MainActivity`

```
public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        EdgeToEdge.enable(this);
        setContentView(R.layout.activity_main);
        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main), (v, insets) -> {
            Insets systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars());
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom);
            return insets;
        });


        Button button = findViewById(R.id.but111111);
        button.setOnClickListener(new MyClick(MainActivity.this));

    }
}

class MyClick implements View.OnClickListener {

    private final Context context;

    public MyClick(Context context) {
        this.context = context;
    }

    @Override
    public void onClick(View v) {
        Log.d("cccccc", "MainActivity.java 处理点击按钮 触发");
        Toast.makeText(context, "MainActivity.java 处理点击按钮 触发", Toast.LENGTH_SHORT).show();
    }
}
```

## strings.xml

但是其实这里是不妥当的  对于安卓开发 字符串应该保存在 string.xml中 并且通过id访问

`res/value/string.xml` 这里就是我们保存的地方

我们增加一个id为text的内容 

```
<resources>
    <string name="app_name">demo</string>
    <string name="text">demo</string>
</resources>
```

那么我们在xml文件中使用的时候 就是通过下面的方法`@string/text`

```
    <TextView
        android:id="@+id/textView"
        android:layout_width="213dp"
        android:layout_height="123dp"
        android:text="@string/text"
        tools:layout_editor_absoluteX="120dp"
        tools:layout_editor_absoluteY="150dp"
        tools:ignore="MissingConstraints" />

```

如果在代码中的话

通过`R` 来访问 这个`R`是在编译后才会产生的 这里只是编译器允许我们提前操作

```
 text.setText(R.string.text);
```

那么以后对字符串的修改 就只需要在strings.xml中操作即可 便利了以后的操作
