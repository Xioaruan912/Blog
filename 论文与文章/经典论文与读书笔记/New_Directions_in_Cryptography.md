New Directions in Cryptography 密码学的新方向

# 摘要

![image-20260527212308884](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260527212308884.png)

说明本文探讨了密码学当代的两个发展

那么这个发展是随着广泛使用的 远程应用APP出现的，需要一个新的加密系统

下面说明了这个paper所需要解决的核心问题：

1. 尽最大减少对于密钥分发通道的需要和提供了与书面签字等同的效能
2. 讨论交流和计算的理论，并且开始提供工具去解决密码学长期所存在的问题

# 介绍



![image-20260527212745542](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260527212745542.png)

这里首先说明了本论文在笔者的地位，是一个对密码学的改革

首先介绍一下当代的计算背景：

```
廉价的数字硬件解放了机械计算的问题，将高等级密码设备的成本降低到可以在此类商业应用中使用的地方，作为远程的取款机和电脑终端

那么一些APP 需要一个新的密码加密系统的出现，尽最大减少对于密钥分发通道的需要和提供了与书面签字等同的效能

同时对于信息理论的理论性发展和计算机科学提供可证明安全的密码系统的承诺，把密码学从古老的艺术，转化为了现实的学科
```

这里就是说明了 当代随着硬件的廉价和APP的出现，需要新的密码方案和对理论的发展

```
计算机系统的发展，让世界两端的人通信和联系变得轻松和廉价，随之带来的就是对抗窃听者和非法消息注入问题，但是当下的密码学完全滞后于其他的通信学科，现代密码学无法满足，就会让用户使用上出现不方便的内容，这样就会消除很多对于远程应用有意义的内容
```

后面说明了当下密码学需要面临的问题 也就是隐私问题

也就是防止未认证的人 可以从不安全的通信信道中 提取到信息，所以我们需要使用密码学来保障隐私

## 密钥交换

这里开始说一下 之前使用加密是如何进行的，需要使用物理快递或者私人会议的方式约定密钥，才可以在远程应用上进行通信

接着说明这个paper的第三部分 提出了一个用于解决密钥交换的方法，使用D和E两个密钥，并且从D计算E是不可行的，把D作为公开密钥放出给公众，使用E作为私钥保存至本地，只有私钥可以计算出自己的消息，这样双方就不再需要约定密钥，而是直接使用对方公开的密钥进行加密消息、

这个时候 第三方窃听者在得到密文的情况下获取消息是完全不可行的，这样就实现了密钥交换

这里可以发现提出了 密钥交换的核心思想

## 签名问题

接着提出了签名问题，希望使用 远程的电子合同签名代替纸质签名方法

合同就是 一个签发出来后，可以给所有人查看和验证，但是不允许修改的内容

提出合同问题就是一个单向认证问题

并且说明了如何将任何公钥密码系统转化为单向认证系统。

# 传统密码学

密码学被认为是数学理论系统，主要解决是隐私和认证问题

隐私问题就是 发送者发送的信息 需要通过不安全的信道 传递给 符合预期的接收者

首先我们定义一个公开信道：如果通信过程是不安全的，那么我们就说明这个信道是公开的

公开信道总会遇见两个问题：窃听者和非法消息的注入
