# 抓包原理

在学习抓包之前 我们先了解一下抓包原理

#### 为什么使用VPN

我们使用VPN 代替 系统代理 抓包 是因为很多APP 不走HTTP 流量 那么可能不走系统代理 从而抓不到包

#### 配置

对于系统代理 我们需要手动 去设置 那么VPN 只需要一键点击就可以开启 代理

#### 抓包范围

对于VPN 可以抓取到 TCP UDP 的数据包  VPN抓包层处于网络层 可以抓取上层所有协议

但是系统代理 没办法抓取

#### 正常通信

非对称加密算法  出名的是RSA和数字签名相关 传输速度慢 加密速度慢

对称加密算法 出名的是AES DES 3DES 传输速度快 加密速度快

那么我们实际传输过程如下：

1. 浏览器访问baidu.com 服务器 返回证书
2. 证书中包含baidu的RSA公钥 并且在本地生成一个随机 对称加密的密钥
3. 通过RSA公钥 加密本地 对称加密的密钥 传输给服务器 实现密钥交换
4. 后续基于 对称加密算法 传输数据

![image-20260326102601224](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260326102601224.png)



#### 解密SSL

那么我们 抓包工具是通过 自签证书 实现安全连接 比如我通过`fiddler `抓包

![image-20260326102813371](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260326102813371.png)

可以发现 颁发者变为了 抓包工具 

那么这样 浏览器以为是在和服务器沟通 实际上数据加解密都是在 抓包工具

然后抓包工具再代替浏览器 去和正常网页进行交互

从而实现解密SSL

可以发现 其实没有什么解密 只是添加了中间人攻击

#### 浏览器能否分析证书

浏览器无法区分 出 正确和错误的实体证书 因为浏览器是面向大众开发的 不可能针对性处理

#### 浏览器如何校验证书

那么我们还需要了解一下如何识别的

主要是识别证书路径和链

![image-20260326103143767](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260326103143767.png)

理论上自签证书是不安全的 但是记得我们抓包证书是存入 系统证书中的

那么系统证书 默认是安全的 所以就可以实现抓包

#### 如何检测

那么如果APP和服务都是一个团队开发 那么可以通信过程中 可以检测是不是自己的服务器的

从而就会导致证书失效 也就是双向验证

用户证书：`/etc/security/cacerts`

# r0capture

这个是一个Hook抓包脚本 原理就是HOOK 了 SSL 的系统函数的一个脚本

那么这个脚本主要是 在加密前 获取数据  所以这个是一个主流的APIHOOK

```
https://github.com/r0ysue/r0capture
```

我们把项目下下来即可 安装依赖

```
pip install loguru
pip install click
pip install hexdump
pip install frida-tools
```

那么工作过程 主要有 附加的方式 

这个模式主要是 你打开了进程 然后

```
python3 r0capture.py -U 连接USB设备抓包 包名 -v -p 生成到哪个pcap文件
```

 还有spawn模式 这个是`frida`启动APP 从启动开始自动抓包

```
python3 r0capture.py -U -f 包名 -v
```

并且需要配合`frida`抓包

这里不演示了 因为比较麻烦 不如`小黄鸟`和` Fiddler`
