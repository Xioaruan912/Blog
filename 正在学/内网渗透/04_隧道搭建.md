# 正向代理和反向代理

这也是面试会提问到的

正向代理就是我们通过`proxy` 代替我们去访问目标网站 并且把返回数据 返回给我们 

也就是我们之间存在一个中间件

![image-20260323120832451](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260323120832451.png)

那么基于这个过程 如果我们得到了一个`中间webshell `那么可以构建正向代理 让服务器代替我们去访问内网其他服务 从而实现渗透

反向代理就是我们URL的实现方案 把 服务器反向代理到URL上 当用户访问URL的时候 不知道具体访问的是哪个服务器 有可能是最近的服务器 自己响应请求 也可能不是 `nginx`就是最常用的反向代理服务器

![image-20260323121140849](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260323121140849.png)

那么上面两个概念在渗透测试中具体的方式有什么呢

1. 正向代理： 我们得到一个具有内网的shell，建立正向代理服务在这台机器上 ，然后我们设置本地网络环境，这样我们对内网其他服务器的访问就没有受到限制
2. 反向代理：我们得到一个具有内网的shell，在上面建立一个反向代理服务，我们需要访问什么数据告知给shell后，木马机器主动连接我们的VPS发送数据（可以绕过只对访问限制的环境中）

那么说完上面的内容 我们说一下如何搭建内网隧道

# proxychains-ng

首先我们安装`proxychains-ng`

```
sudo apt update
sudo apt install proxychains4
```

配置默认是在`/etc/proxychains4.conf`

里面存在这个 修改如下

```
[ProxyList]
# add proxy here ...
# meanwile
# defaults set to "tor"
socks5 	0.0.0.0 1080
#socks4 	0.0.0.0 1080
#socks4 	0.0.0.0 1080
```

那么就是构建完毕了一个` sock5`代理（这里是通过本地的socks5代理 真实情况是修改为木马机器的IP）

只要我们在目标木马机器构建了代理 那么就可以通过木马机器访问
