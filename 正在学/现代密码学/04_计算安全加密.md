这里我们依旧定义一下计算安全加密

首先我们说一下 对于私钥算法的语法，其次我们允许解密算法 输出异常

我们还让M空间 作用于所有长度有限的二进制字符串的集合，也就是任意长度的二进制字符串都可以是我们的消息

# 定义

私钥加密方案由3个 **概率性多项式时间算法**组成

### Gen（密钥产生算法）

1^n 是安全参数的**一进制（Unary）表示形式**

按照下述公式生成Key
$$
k \gets Gen(1^n)
$$
也就是说 输入给Gen算法 需要的安全参数，那么Gen就会给出对应强度的Key

我们允许Gen算法是随机算法

### Enc（加密算法）

输入Key和Message 并且Message是由有限长度的二进制字符串组成，并且输出密文c
$$
c \leftarrow Enc_k(m)
$$

### Dec（解密算法）

输入密文 和Key 输出的是明文M或者是Error ，对于Error 我们使用符号 ⊥表示

我们的算法当然还需要保持下述数学基础
$$
Dec_k(Enc_k(m)) = m 
$$
如果我们的M空间 不是有限长度的，而是一个固定的 那么我们又引出一个密码方案 

```
a ﬁxed-length private-key encryption scheme for messages of length l(n).
```

消息固定长度为l(n)的私钥密码方案

# 私钥密码体系的基本安全定义（EAV-安全）

对于私钥密码体系，给定的一个密钥，只可以加密于一个单一消息

对于以前所有的定义 我们都需要一个 威胁模型和 一个安全目标

这里我们都设定基本的场景

1. 攻击者只是监控信道，并且获得单一的密文
2. 攻击者只是具有受限计算资源并且需要在有效时间内攻击成功 那么才算攻击成功
3. 我们不对攻击者的攻击策略规定

这又引出了一个关键的特点

```
the deﬁnition ensures protection against any computationally bounded eavesdropper, regardless of the algorithm it uses
```

我们对于密码安全的评估 是看有限计算资源的，但是无论使用何种攻击算法的攻击者

那么对于我们的防守目标就是下述内容

```
the adversary should be unable to learn any partial information about the plaintext from the ciphertext.
```

攻击者无法从密文中得到明文的任意信息——语义安全

那么这个安全保障我们很难实现 但是我们又存在一个不可区分性的定义 等同，这个定义就简单很多了

# 窃听者环境下私钥加密实验（修改）

我们之前存在一个实验——窃听者环境下私钥加密实验
$$
\text{PrivK}_{A,\Pi}^{\text{eav}}
$$
实验过程大概如下：

1. 攻击者A选择两个消息 m1 m2
2. Gen生成一个对应安全参数n的Key
3. 实验随机抛出硬币，选择加密哪个 记作b
4. 通过Enc生成挑战密文c 发送给攻击者A
5. 攻击者依据c猜测 输出一个b'
6. 如果 b'=b 那攻击成功 否则失败

那么之前的知识点我们回顾完毕 这里我们保持 这个实验不动 但是加入重要的修改对于实验

1. 攻击者需要在 有限时间内猜测
2. 攻击者攻击成功的概率比 1/2略好
3. 攻击者需要选择两个长度一样的消息作为实验的开始

那么这里我们又得到了一个基本的概念
$$
\Pr[\text{PrivK}_{A,\Pi}^{\text{eav}} = 1] \leq \frac{1}{2} + \text{negl}(n)
$$
对于攻击成功的概率我们需要保持在大约 1/2 + 可忽略概率

由于这里我们加入了安全参数n 所以实验我们也可以修改为下述标志
$$
\text{PrivK}_{A,\Pi}^{\text{eav}}(n)
$$
那么下面我们将对这个实验进行 正式的定义

符号 Π = (Gen, Enc, Dec)

攻击者记作 A

1. A输入一个 1^n和一对长度固定的消息 m1 m2 
2. Gen(1^n) 输出一个Key和选择一个 bit 用于选择加密哪个消息
3. Enc(Key,m1/2) 输出一个密文 交给A 记作挑战密文
4. A依据挑战密文选择一个 b'
5. 如果 b' = b 那么实验输出 1 记作攻击成功 否则 失败

![image-20260424162734196](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260424162734196.png)

这个定义 其实比 之前我们的定义 要 弱很多，所以 我们又可以推测出 只要是满足完全保密性的算法 都满足EAV-安全概念

所以我们下面所展示的内容就是 满足这个定义 但是不满足之前 完全保密性的定义的

# 明文长度的说明

实际上 对于所有的加密方案 我们都不可能 不泄露长度信息 无论是RSA AES DES

这是因为 我们不可以依据 随意长度的消息 去隐藏信息

虽然大多数情况 我们泄露明文长度信息是无关紧要的 但是事实并非全是如此

1. 简单数值/文本数据：比如我们需要知道 薪资是5位数还是6位数 又或者我们需要知道结果是 Yes还是No 那么依据长度我们就可以直接判断
2. 自动建议：我们的输入法，可能会自动补全一些信息，那么这个自动补全 就可能会泄露我们常输入的内容
3. 数据库检索：对于返回的记录数 我们又可以得到一些关于数据库的信息
4. 压缩的文件：如果文件可以被压缩 那么说明存在很大部分的 空余空间，如果攻击者可以在 加密前控制 压缩数据那么就可以写入一些木马攻击

对于上述的内容 其实现代密码 普遍使用一个方法来应对：Padding 也就是填充数据

# 语义安全性

## 两个攻击者对于猜测不懂内容的概率提升

我们对于安全加密的定义就是： 攻击者无法从密文中得到任何有关 明文的信息

我们对于 EAV-安全的 概念说的是 不泄露任何 与单个bit有关的信息

![image-20260424165544618](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260424165544618.png)

大白话理解：即使你手里有密文，你想知道 明文的第 3 位是 0 还是 1？ 你的胜算也和抛硬币瞎猜一模一样。

如果你能以 60% 的概率猜对某一位，这个方案就违背了这个定理，也就不再是 EAV-安全的了。

下面是对这个定理的证明

## 证明如下：

我们要攻击成功 ，如果我们可以确定 密文的第i个位置的M 那么也可以确定出 m0 m1

我们使用 归约证明 来证明这个内容

设定 I0 是所有bit=0的集合 I1是所有bit=1的集合

那么猜对的概率如下：
$$
\frac{1}{2} \cdot P(\text{猜测 0 } | \text{第 $i$ 位是 0}) + \frac{1}{2} \cdot P(\text{猜测 1} | \text{第 $i$ 位是 1})
$$
那么这里存在两个攻击者 A' 是用于猜测位数的攻击者 A0是接受挑战密文的攻击者

1. A0 从 I0 和 I1中 抽取 m0 m1 作为加密的输入
2. Gen 按照安全参数n 生成K 并且选择一个 b 用于选择加密
3. Enc 加密数据 记作挑战密文交给A0
4. A0 让A'猜测 此密文对应的明文的第i位是 0 还是1
5. 如果A'猜测是0 那么就A0就得出结果是m0 

那么这个方法 就可以说明 

如果存在A' 猜位的准确率比 1/2 高

那么A0区分的概率也比1/2高

那么我们又可以计算一下 A'的成功概率

![image-20260424171022905](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260424171022905.png)

那么这个之前 我们又有 EAV安全
$$
\Pr[\text{PrivK}_{A,\Pi}^{\text{eav}} = 1] \leq \frac{1}{2} + \text{negl}(n)
$$
由此就可以推出
$$
\Pr[\mathcal{A}(1^n, \text{Enc}_k(m)) = m^i] \leq \frac{1}{2} + \text{negl}(n)
$$
所以就可以得到结论 这个方法在数学上是不可行的

下面我们论证

## 两个攻击者对于密文的概率提升

**无论明文m如何分布任何概率多项式时间的敌手都无法获得有效信息**

这里我们探讨的是 从密文中学习参数 输入f 从而让正确率提升

如果你一直输入的是1 那么就算不用猜测也可以分析出 

定理如下：

![image-20260424171836073](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260424171836073.png)

核心是两个概率的相减
$$
\left| \Pr[\mathcal{A}(1^n, \text{Enc}_k(m)) = f(m)] - \Pr[\mathcal{A}'(1^n) = f(m)] \right| \leq \text{negl}(n)
$$
A是有密文的攻击者  需要猜出关于明文的某个特征 f(m)

A'是没有密文的攻击者 只有安全参数n 但是他是按照 分布概率 D 去猜测

核心讨论点就是 如果攻击者即使知道了 密文 但是获胜率依旧没有提升 那么就满足 EAV安全

## 证明如下：

首先确定我们需要证明的内容

如果系统是EAV安全的 那么说明攻击者无法从 Enc中区分m 在得到分布D和密文C的情况下

首先 A 可以成功通过Enc计算出f(m) ，并且A可以区分出Enc(m)和Enc(1的字符串)

1. 从D中选择m 输出 m0 = m m1 = 1
2. 选择一个进行加密
3. 如果A输出的是f(m) 那么实验输出0
4. 如果A 对于 m和1的输出概率明显存在区别

那么上述实验过程 就说明了 这个算法 是不存在 EAV安全的 这里是通过反证法实现的

对于A' 是只有 D的信息也去计算f(m)和A一样 他其实就是 把需要分析的内容交给A去判断

然后直接复读A的结果

那么其实就证明了 就算我给出一个虚假的加密数据 给A 但是他分析半天 连真假都分辨不出来

# 总结

这里我们学习了两个概念 一个是 **不可区分性** 另一个是 **语义安全**

语义安全就是：即使存在一些附加信息（允许猜测位、存在密文、存在提示、泄露长度） 都无法得到与明文有关的任何信息（与看空气差不多）

不可区分性：二选一的游戏中 猜测不出

并且我们证明了 两者是等价的 

现实情况中 语义安全是很难应用的 所以我们可以通过不可区分性 的判别 推测出 语义安全
