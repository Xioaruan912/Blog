## `添加串行端口`

![image-20260317221013723](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221013723.png)

修改为通过管道

![image-20260317235207879](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317235207879.png)

继续开机 我们通过CMD 继续配置 通过管理员打开CMD

```
bcdedit /copy {current} /d "Debug"
bcdedit /timeout 10
```

这两个命令是 构建一个全新的引导  输入完毕后 重新启动

![image-20260317221026687](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221026687.png)

进入` Debug` 然后再继续 我们通过` msconfig` 检查是否进入的是`Debug`

![image-20260317221041926](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221041926.png)

然后继续 管理员打开CMD

```
bcdedit /dbgsettings serial debugport:1 baudrate:115200
bcdedit /bootdebug On
bcdedit /debug on
```

![image-20260317221051450](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221051450.png)

输入完毕后我们可以发现 这里的调试选项已经打开了 下面都是可以添加可以不添加的

```
bcdedit /set testsigning on
bcdedit -set loadoptions DDISABLE_INTEGRITY_CHECKS

bcdedit /enum
bcdedit /dbgsettings
```

如果配置成功 重新启动 就会进入黑屏

这就代表配置成功了 进入了调试模式 等待需要有人调试 也就是我们的`WinDBG`

# WinDBG

## 新版

我们安装好虚拟机后 就需要通过`WinDBG`开始调试 微软出品 微软商店下载即可

```
https://learn.microsoft.com/en-us/windows-hardware/drivers/debugger/
```

存在新和旧版本 旧版本安装WDK后默认安装

我们去设定一个文件夹`Symbols `这个是符号的意思 也就是为了让我们 通过符号进行逆向分析 在`WinDBG `首页的`settings`

![image-20260317221111190](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221111190.png)

```
srv*E:\Symbols*http://msdl.microsoft.com/download/symbols
```

如果需要系统代理 我们就需要添加环境变量

```
_NT_SYMBOL_PROXY
```

![image-20260317221130777](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221130777.png)

开始链接调试 点击左上角文件

![image-20260317221138523](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221138523.png)

点击到`Com`

![](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221138523.png)

点击oK后如果虚拟机启动的话 可以提示

![image-20260317221150424](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221150424.png)

我们对命令行输入 `g` 只要进入操作系统之前 有断点就直接`g` 如果发现 `BUSY` 那么就是在下载或者在执行 等待即可 我们可以通过一些 命令进行测试

我们需要手动打断点

![image-20260317221157129](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221157129.png)

然后执行下述内容

```
!process 0 0
!idt
```

如果显示下述内容 就是说明 双机调试应该没有问题了

![](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260317221157129.png)

## 老版本

如果我们使用老版本 那么需要通过快捷方式 后面添加上

```
-b -k com:pipe,port=\\.\pipe\com_1,baud=115200,reconnect -y srv*E:\Symbols*http://msdl.microsoft.com/download/symbols
```

启动即可
