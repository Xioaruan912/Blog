之前我们学习的 都是 一个密文 在两个可信者之间互相传送，那么现在我们需要回归实际情况了

也就是多密文的安全性情况：

双方发送多条密文 并且使用同一个KEY加密，这个也是我们现实中需要关注的情况

# 多密文情况的安全定义

在对这个场景分析之前 我们首先对其下定义，我们依旧使用的是 私钥加密窃听实验

1. A 输入 安全参数 1^n 与两个 消息集合M 其中 m0  = {m01,m02,m03,....} m1 = {m11,m12,m13....}，并且要求其长度全部一样
2. 使用Gen(1^n) 产生KEY 并且生成一个 b 用于选择 加密m0 还是 m1(集合里面的消息全部使用一个KEY加密)
3. 使用Enc加密后发送给A A分析并且输出 b'
4. 如果b = b' 则实验输出1 否则输出0

那么我们下面给出安全的定义

总的来说 只要 Pr[实验成功] 小于等于 1/2+可忽略概率，那么我们就i说明这个加密方案具有多加密安全性

那么我们下面提出一个命题：

## 是否存在一个密码方案存在单密文安全但是多密文不安全的情况呢？

我们下面对其进行论证

我们使用的是 一次一密进行，并且通过实验证明其存在

我们对A的消息设置进行假设 m0 = {0000,0000} m1 = {0000,1111}

依据实验进行加密 得到 C = {c1,c2} 交给A进行判别并且输出b'

那么这里我们其实可以发现 我们只需要分析 c1 ?= c2 即可输出对应的b'=b

由于使用同一密钥进行加密 那么对于相同的消息 密文也是一样的

所以这里就说明了 使用一次一密 对多消息加密的密码方案是不安全的 

# 概率加密的必要性

上面我们知道了 使用 同一个KEY 对多密文进行加密 无法保障密码方案的安全性

换句话来说：只要我们的 加密算法是 无状态并且确定的 就无法保障我们多密文情况下的安全，所以产生了下述定理

**只要密码方案的加密算法 使用的是 对于KEY和Message确定的函数，就无法保障我们的多密文同时加密的安全性**

我们不可以认为是这个定理过于强大，而是应该认为这是我们目前学习的密码体系中的一个安全漏洞

那么解决这个问题 我们需要设定一个加密方案，并且其加密函数具有随机化的功能，在同时对不同明文加密的时候，可以产生不同的密文

# 选择明文攻击

选择明文攻击情况是攻击者可以控制可信双方发送的明文

在可信双方交换密钥的时候，使用KEY对M进行加密，但是攻击者可以观察到，并且知道某一个明文是被控制加密的

比如存在两个消息m1 m2（攻击者全都知道），那么对于这个攻击场景 使用KEY对m1 m2加密成密文 并且传输

攻击者可以窃听并且获取到

那么对于选择明文攻击的安全性保障的意思是：

即使在上述场景中，攻击者获取到了 密文C={c1,c2} 但是依旧无法分辨出 哪个是m0 m1，即分辨出的概率不会大于瞎猜

# 现实中的选择明文攻击

现实中也是存在选择明文攻击的手段的

比如 攻击者得到一个终端的控制，这个终端是和另一个可信方的通信，那么这个时候就成功得到了 选择明文攻击（即使不知道KEY），那么我们保障的安全性就是说 即使在这个情况下，依旧无法得到有用的信息

其实选择明文攻击也包括已知明文攻击，不需要操作 可信双方的控制，而是依据已知信息的攻击手段

这个攻击手段在二战中也使用到了

英国埋下地雷，并且知道了地雷的精确坐标，德国也发现了地雷，但是通过加密方法加密坐标发送给总部，那么英国就是通过已知明文攻击 破解了加密通信

日本攻打中途岛，设定AF为名字，但是美国不知道，所以美国发送虚假信息，说明中途岛没水资源了，日本发送AF没水资源，从而破解并且阻止了这个攻击情况

# CPA安全

这里的CPA安全 其实就是 选择明文攻击的安全性

那么这里的实验定义就是 

给出一个加密方案给攻击者，但是这个加密方案是一个黑盒（也就是攻击者不知道加密的KEY和细节）

我们想象成攻击者有权限访问这个加密算法Enc，攻击者可以自行输入A 并且获得密文（如果加密算法是随机的，那么密码方案在每次加密后都随机生成）

所以我们产生了 对于CPA安全的测试实验
$$
 PrivK_{A, \Pi}^{cpa}(n)
$$


1. Key被 Gen(1^n)产生
2. 攻击者A给定输入1^n 并且有权限操作Enc，随后可以输出一对相同长度的 m0 m1 作为明文
3. A选择一个b 决定加密哪个明文
4. A猜测是哪个明文 并且输出b'
5. 如果b'=b 那么结果输出1 否则输出0

由此实验 我们就可以得到 私钥密码方案满足CPA安全的定义：

```
一个私钥加密密码方案如果在选择明文攻击中具有不可区分性，或者存在CPA安全，并且对于所有的PPT攻击者A，都存在一个可忽略函数negl满足下述：
```

$$
P[PrivK_{A, \Pi}^{cpa}(n) = 1] \leq \frac{1}{2} + \text{negl}(n)
$$

# CPA安全对于多密文情况

我们其实对上述实验只需要切换回 List方法 即可替换为多密文情况

但是这里我们会变为稍微简单一点的内容

赋予攻击者访问一个 `左或右终端`的权限 记作：
$$
LR_{k.b}
$$

1. 输入一对消息m0 m1  并且等长
2. 并且通过实验产生一个b 如果b=0 那么攻击者A收到的全是左侧明文 如果b=1 那么攻击者A收到的是右侧明文
3. 攻击者开始猜测b是什么

这里我们就引出了 另一个实验方案
$$
PrivK_{A, \Pi}^{LR-cpa}(n)
$$

1. Gen(1^n)产生KEY
2. 选择一个b
3. 攻击者给定输入 1^n 并且有权限访问 LR终端
4. A输出一个b'
5. 如果b'=b 那么就成功 并且输出1 否则输出0

那么基于上述实验我们又可以得到一个 对于多密文情况的CPA安全定义

```
一个私钥密码方案在多密文情况下具有不可区分性或者具有多密文的CPA安全，也就是说存在一个可忽略函数满足下述：
```

$$
P[PrivK_{A, \Pi}^{LR-cpa}(n) = 1] \leq \frac{1}{2} + \text{negl}(n)
$$

这里我们会很好奇 为什么需要左右呢 其实出现左右就可以在多密文情况下 模拟一个加密终端

CPA安全 == 基于多密文的CPA安全

所以我们又有下述定理：

![image-20260501103836449](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260501103836449.png)

私钥加密方案 只要有CPA安全 那么也具有多密文CPA安全

所以对于一个私钥加密方案 我们只需要证明其存在 CPA安全

## 固定长度和可变长度Message

其实 我们给出一个具有 CPA安全的固定长度加密算法后，可以很简单的给出一个具有CPA安全的可变长度加密算法

比如 固定长度具有 1bit的Message长度的密码加密方案具有CPA安全

那么 如果传递的是 nbit 只需要每个bit都按照上述加密方案加密后 即可得到一个多密文情况下的CPA安全
