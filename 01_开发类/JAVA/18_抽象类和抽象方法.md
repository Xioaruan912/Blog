在之前编写代码过程中

我们知道 如果 子类 需要完成 `eat` 那么 父类必须写一个 没有意义的 `eat` 通过子类重写实现

那么这就会 很麻烦 所以导出了抽象方法

```
public abstract void eat();
```

如果一个类中 出现了 抽象方法 那么 这个类 必须要转化为 抽象类

```java
package src;

public abstract class person {
    public  int age = 00000;
    public  abstract void eat();
}

```

并且 所有 子类 必须 重写  此抽象方法 并且 抽象类 `不允许创建对象` 只可以被`继承`
