# `static`关键字 - 成员

我们在面向对象的 方法中 不加入 `static` 关键字

如果我们构造 一个 `Student 类` 其中的` teacher`成员变量 是一样的  

那我们需要 初始化的时候 给每个 都赋值一样的` teacher`

`static修饰成员变量`：被这个类的所有对象共享，有一个修改后全部都修改

```java
package src;

public class Student {
    private String name;
    private int age;
    static String teacherName; //所有类 共享一个teacher
}
```

那么我们通过下面方法

```java
package src;

public class Main{
    static void main(String[] args) {
        Student d2 = new Student();
        Student.teacherName = "大老师"; // 为这个类的teacher 设置为"大老师"
        d2.name = "xiaoming";
        d2.age =11;
    }
}
```

# `static`关键字 - 方法

我们之前知道 测试类 也就是`main` `javabean类` 也就是一个对象

主要操作 有下面几个：

## 见名知意

```
public class ArrayUtil{

}
```

我们需要通过 名字 从而输出 我们这个类具体是干嘛的

## 私有化构造方法

我们不需要声明 对象 所以我们直接构造方法私有化

```
public class ArrayUtil{
	private ArrayUtil(){}

}
```

## 所有方法设置静态 并且通过类名字调用

```
public class ArrayUtil{
	private ArrayUtil(){}
	public static int getMax(){...}
	public static int getMin(){...}
	public static int getSum(){...}
	public static int getAvg(){...}
}
```

# 注意事项

1. 静态方法只能访问静态变量或者其他的静态方法
2. 非静态无所谓
3. 静态方法没有`this`关键字

# Final关键字-变量

可以修饰 变量 类和方法 表示最终不可变的

如果对变量使用final关键字 那么后续无法进行任何修改

```
final int a = 10;
```

那么后续无法对其进行 修改 变为常量 一般写成大写

如果对 基本数据类型 那么是 变量里面的数据无法修改

如果对 引用数据类型 那么是 变量里面的地址无法修改

# 枚举

如果约束对象个数 需要只能构建 有限个 那么就需要枚举

定义格式

需要私有化构造方法

```
public enum 枚举类名字{
	枚举对象名字(),
	枚举对象名字(),
	枚举对象名字(),
}
```

![image-20260129235023254](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260129235023254.png)

所有的枚举对象 都是通过 `public static final` 定义的

# 枚举的注意事项

 第一行必须 定义好枚举的对象 并且通过 逗号隔开分号结尾

枚举类的构造方法 必须是 `private`

编译器会默认给枚举构建两个构造方法 `values()` `valuesOf()`
