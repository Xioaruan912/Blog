# 面向对象的数据安全问题

![image-20260120160041810](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260120160041810.png)

可以发现 这里是有问题的

所以我们引入 `private`关键字 用于修饰 成员变量和成员方法

一旦被 `private` 修饰 那么只可以在本类中被访问 外界无法访问

![image-20260120160221586](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260120160221586.png)

这样就会报错

我们可以通过构造 `set(赋值)` ` get(取值) ` 两个方法 对私有变量操作

```java
package src;

public class Dog{
    String name;
    private int age;
    public void setAge(int num){
        if(num >0 && num <100){
            age = num;
        }
    }
    public int getAge(int num){
        return age;
    }
}
```

这样我们在测试类中 就需要通过下面格式修改

```java
package src;

public class Main{
    static void main(String[] args) {
        Dog d = new Dog();
        d.name = "小明";
        d.setAge(11);   
    }
}
```

如果 赋值错误 那么就 返回的是 `java的默认值`

# this关键字

![image-20260120160814242](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260120160814242.png)

如果我们需要传递的参数和 成员变量的参数名字一样 那么我们就需要通过 `this` 关键字  区分 是类里面的还是类外的

```java
package src;

public class Dog{
    String name;
    private int age;
    public void setAge(int age){
        if(age >0 | age <100){
            this.age = age;
        }
    }
    public int getAge(int age){
        return this.age;
    }
}
```

如果我们一定要都通过`age` 那么就会通过就近原则 所以方法会通过局部变量 执行

# 构造方法

对于我们希望 `new`一个对象的时候 直接赋值 就需要通过构造方法实现

```
修饰符 类名字(参数1){
	方法体
}
```

1. 需要和类名字一样的方法名字
2. 可以带空和全部参数的构造方法
3. 可以定义多个构造方法 只是参数个数不同的问题 也就是方法的重载

```java
package src;

public class Dog{
    private String name;
    private int age;

    public Dog(){
        System.out.println("111111");
    }
    public Dog(String name,int age){
        this.age = age;
        this.name = name;
    }
}

```

在测试类中 我们可以通过 下面调用不同的构造函数

```java
package src;

public class Main{
    static void main(String[] args) {
        Dog d = new Dog("小明",11);
        Dog d2 = new Dog();
    }
}
```

# 构造方法的注意事项

1. 如果不写构造方法 那么系统默认提供 不带参数的构造方法
2. 如果我们自己定义出来了 构造方法 那么系统就不提供了
3. 以后无论使用与否 都写出 带参数和不带参数的构造方法
