双方发送多条密文 并且使用同一个KEY加密，这个也是我们现实中需要关注的情况

# 多密文情况的安全定义

在对这个场景分析之前 我们首先对其下定义，我们依旧使用的是 私钥加密窃听实验

1. A 输入 安全参数 1^n 与两个 消息集合M 其中 m0  = {m01,m02,m03,....} m1 = {m11,m12,m13....}，并且要求其长度全部一样
2. 使用Gen(1^n) 产生KEY 并且生成一个 b 用于选择 加密m0 还是 m1(集合里面的消息全部使用一个KEY加密)
3. 使用Enc加密后发送给A A分析并且输出 b'
4. 如果b = b' 则实验输出1 否则输出0

总的来说 只要 Pr[实验成功] 小于等于 1/2+可忽略概率，那么我们就i说明这个加密方案具有多加密安全性

## 是否存在一个密码方案存在单密文安全但是多密文不安全的情况呢？

我们下面对其进行论证

我们使用的是 一次一密进行，并且通过实验证明其存在

我们对A的消息设置进行假设 m0 = {0000,0000} m1 = {0000,1111}

依据实验进行加密 得到 C = {c1,c2} 交给A进行判别并且输出b'

那么这里我们其实可以发现 我们只需要分析 c1 ?= c2 即可输出对应的b'=b

由于使用同一密钥进行加密 那么对于相同的消息 密文也是一样的

所以这里就说明了 使用一次一密 对多消息加密的密码方案是不安全的 

# 概率加密的必要性

上面我们知道了 使用 同一个KEY 对多密文进行加密 无法保障密码方案的安全性

换句话来说：只要我们的 加密算法是 无状态并且确定的 就无法保障我们多密文情况下的安全，所以产生了下述定理

**只要密码方案的加密算法 使用的是 对于KEY和Message确定的函数，就无法保障我们的多密文同时加密的安全性**

我们不可以认为是这个定理过于强大，而是应该认为这是我们目前学习的密码体系中的一个安全漏洞

那么解决这个问题 我们需要设定一个加密方案，并且其加密函数具有随机化的功能，在同时对不同明文加密的时候，可以产生不同的密文

# 选择明文攻击

选择明文攻击情况是攻击者可以控制可信双方发送的明文

在可信双方交换密钥的时候，使用KEY对M进行加密，但是攻击者可以观察到，并且知道某一个明文是被控制加密的

比如存在两个消息m1 m2（攻击者全都知道），那么对于这个攻击场景 使用KEY对m1 m2加密成密文 并且传输

攻击者可以窃听并且获取到

那么对于选择明文攻击的安全性保障的意思是：

即使在上述场景中，攻击者获取到了 密文C={c1,c2} 但是依旧无法分辨出 哪个是m0 m1，即分辨出的概率不会大于瞎猜

# 现实中的选择明文攻击

现实中也是存在选择明文攻击的手段的

比如 攻击者得到一个终端的控制，这个终端是和另一个可信方的通信，那么这个时候就成功得到了 选择明文攻击（即使不知道KEY），那么我们保障的安全性就是说 即使在这个情况下，依旧无法得到有用的信息

其实选择明文攻击也包括已知明文攻击，不需要操作 可信双方的控制，而是依据已知信息的攻击手段

这个攻击手段在二战中也使用到了

英国埋下地雷，并且知道了地雷的精确坐标，德国也发现了地雷，但是通过加密方法加密坐标发送给总部，那么英国就是通过已知明文攻击 破解了加密通信

日本攻打中途岛，设定AF为名字，但是美国不知道，所以美国发送虚假信息，说明中途岛没水资源了，日本发送AF没水资源，从而破解并且阻止了这个攻击情况
