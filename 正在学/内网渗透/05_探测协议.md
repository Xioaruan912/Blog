# 禁止出站IP

这里的意思就是 目标机器只允许主动访问公网指定的IP 这样就没有办法反弹shell 了

也就是只可以访问白名单的IP，我们没有办法伪造

# 禁止出站端口

也就是可以访问公网的所有IP 但是只允许访问特定端口 比如80，443等

这样我们需要去进行探测

## linux自带命令

对于linux 我们可以直接使用shell命令构建探测

```
for i in {443..449}; do timeout 0.5 bash -c "echo >/dev/tcp/baidu.com/$i" && echo "$i open" || echo "$i closed";done
```

上述脚本就是 通过端口的循环 访问 看看百度是否访问成功 

探测目标机器的端口是否开放出站

![image-20260323125251778](https://raw.githubusercontent.com/Xioaruan912/pic/main/image-20260323125251778.png)

或者写入文件中

```
for i in {443..449}; do timeout 0.5 bash -c "echo >/dev/tcp/baidu.com/$i" && echo "$i open" || echo "$i closed";done >> res.txt
```

## windows

如果是windows 那么我们需要先对抗杀软 然后释放工具探测即可 这个就不需要继续展示了



## 攻击机器

那么我们攻击机器可以设置全端口转发方法 从而进行探测

通过 下述命令 就可以让 VPS所有的访问 全部送入 34444端口

```
iptables -A PREROUTING -t nat -p tcp --dport 1:65535 -j REDIRECT --to-port 34444
```

我们只需要配合`nc`监听即可

```
nc -lvp 34444
```

下面是基本命令

```
iptables -t nat -nvL 		//查看规则
iptables -t nat -F 			//清空规则
```

# 禁止协议

有可能目标内网机器只允许某个协议出网 我们还需要进行探测

## ICMP协议

我们直接使用 ping命令 即可知道是否允许icmp出网

```
ping www.baidu.com
```

如果有回显 说明DNS协议和ICMP 都可以出网

否则我们直接 对ip单独探测即可

```
ping 183.2.172.177
```

## DNS协议

可以使用`nslookup` `ping` `dig` 等命令

```
ping www.baidu.com
nslookup www.baidu.com
dig www.baidu.com
```

## HTTP协议

我们可以使用 `curl` `wget` 等方法 查看是否能够访问

```
curl http://baidu.com
```

windows使用如下内容

```
certutil -urlcache -split -f http://baidu.com
bitsadmin /transfer test http://baidu.com c:\1 						//构建下载任务
powershell iwr -Uri http://baidu.com -OutFile 1 -UseBasicParsing
```
